# 精力管理领域知识

## 目的

本文档为 lxp-eng-planning（吕小鹏精力规划系统）提供精力量化模型、任务分解模型与预警机制的领域约束定义。解决的核心问题是：如何将个人精力从抽象概念量化为可计算、可预警的数值体系。

## 适用角色

- 需求文档管理者（编写需求规格时参考领域定义）
- 技术文档管理者（设计技术方案时遵循领域约束）
- 代码生成者（实现业务逻辑时遵守领域规则）

---

## 1. 精力量化模型定义

### 1.1 三档标准

| 精力等级 | 标签 | 预估耗时范围 | 精力点（energy_points） | 典型任务示例 |
|---------|------|-------------|----------------------|------------|
| 高 | high | ≥ 4 小时 | 4.0 | 从零搭建项目框架、深度学习新领域 |
| 中 | medium | 2 ~ 4 小时（含 2，不含 4） | 2.0 | 完成一章教程并练习、编写核心模块代码 |
| 低 | low | < 2 小时 | 1.0 | 回复邮件、整理笔记、简单阅读 |

### 1.2 边界归属规则

| 耗时 | 精力等级 |
|------|---------|
| 恰好 2 小时 | **中（medium）**——包含下界 |
| 恰好 4 小时 | **高（high）**——包含下界 |
| < 2 小时 | 低（low） |
| 2 ~ 4 小时（不含4） | 中（medium） |
| ≥ 4 小时 | 高（high） |

### 1.3 存储字段

```
energy_level:  枚举值 HIGH / MEDIUM / LOW
energy_points: 派生浮点数 4.0 / 2.0 / 1.0（由 energy_level 映射）
```

## 2. 精力预算定义

### 2.1 每日精力预算（daily_budget）

| 属性 | 值 |
|------|-----|
| 默认值 | **8.0** 精力点（对应约 8 小时有效投入） |
| 可自定义范围 | 4.0 ~ 16.0 |
| 步长 | 0.5 |
| 设置位置 | 用户设置页面 |

### 2.2 每周精力预算（weekly_budget）

| 属性 | 值 |
|------|-----|
| 默认值 | **56.0** 精力点（每日预算 8.0 × 7 天，含周六周日） |
| 可自定义范围 | 28.0 ~ 112.0（对应每日预算下限/上限 4.0/16.0 × 7 天） |
| 计算公式 | `weekly_budget = daily_budget × 7` |

**重要**：周预算覆盖**完整 7 天**（周一至周日），不区分工作日和周末。

### 2.3 周末任务处理规则

周六周日与工作日**同等处理**：
- 系统不区分工作日/周末
- 周末排程的任务，其 `energy_points` **正常计入**当日 `daily_load` 和当周 `weekly_load`
- 用户可自行决定是否在周末排程任务
- F2 周计划视图的 7 列（周一至周日）功能完全一致

## 3. 三层任务分解模型

### 3.1 层级定义

| 层级 | level 值 | 名称 | 示例 | 可排程 |
|------|---------|------|------|--------|
| 第 1 层 | 1 | 长期目标 | 学会 Rust | ❌ 不可排程 |
| 第 2 层 | 2 | 短期任务 | 完成 Rust 基础教程第 1-5 章 | ❌ 不可排程 |
| 第 3 层 | 3 | 可执行任务 | 阅读《Rust Programming Language》第 3 章并完成练习 | ✅ 可排程 |

### 3.2 严格限制

- **严格限定 3 层，不可再嵌套**
- level=1 的任务 `parent_id = null`（根节点）
- level=2 的任务 `parent_id` 指向 level=1 任务
- level=3 的任务 `parent_id` 指向 level=2 任务
- **只有 level=3 的可执行任务可以排程到日历**（设置 `scheduled_date` 和 `scheduled_period`）

### 3.3 数据库层约束

```
level 字段：CHECK (level IN (1, 2, 3))
parent_id 关系：level > 1 时 parent_id 必填
scheduled_date：仅 level = 3 时可设置（应用层校验）
```

## 4. 精力负荷聚合规则

### 4.1 当日精力负荷（daily_load）

```
daily_load = 当日所有已排程任务的 energy_points 之和
```

**聚合范围**：当日全部时段（上午 + 下午 + 晚上）所有已排程任务的 energy_points 之和。时段（上午/下午/晚上）仅作为分类标签。

**示例**：
- 上午排了 1 个高（4.0）+ 1 个低（1.0）= 5.0
- 下午排了 1 个中（2.0）= 2.0
- 晚上排了 1 个低（1.0）= 1.0
- **daily_load = 5.0 + 2.0 + 1.0 = 8.0**

### 4.2 当周精力负荷（weekly_load）

```
weekly_load = 当周（周一至周日，共 7 天）所有已排程任务的 energy_points 之和
```

**示例**：一周内排了 10 高 8 中 4 低
```
weekly_load = 4.0×10 + 2.0×8 + 1.0×4 = 40.0 + 16.0 + 4.0 = 60.0
```

### 4.3 显示格式

| 场景 | 显示位置 | 格式 |
|------|---------|------|
| 日计划视图 | 每个时段区域 + 顶部 | `精力点合计 / 每日预算`（如 `8.0 / 8.0`） |
| 周计划视图 | 每列 + 顶部横幅 | `本周精力负荷 X.0 / 56.0` |

## 5. 超载预警机制

### 5.1 阈值定义

```
overload_threshold = weekly_budget（默认 56.0）
```

- `overload_threshold` 随 `weekly_budget` **自动联动**
- 用户自定义 `weekly_budget` 时，`overload_threshold` 无需单独设置

### 5.2 触发条件

```
当 weekly_load ≥ overload_threshold（= weekly_budget，默认 56.0）时触发预警
```

**适用场景**：F2 周视图实时预警 + F4 统计页预警。

### 5.3 预警方式

| 方式 | 实现 |
|------|------|
| 列高亮标记 | 橙色（接近超载，≥90%）/ 红色（已超载，≥100%） |
| 顶部横幅文案 | 「⚠ 本周精力负荷 58.0 / 56.0，已超载，建议调整」 |

### 5.4 日负荷超载提示

当 `daily_load > daily_budget` 时：
- 当日列高亮橙色
- 拖拽时实时显示 `今日已排 2 高 1 中 = 10.0 / 8.0 ⚠ 超载`

## 6. 每日精力消耗报告

### 6.1 生成时机

用户**当日首次打开日计划视图**时，若前一天的报告尚未被查看，则**自动弹出**展示。

- 不依赖后台定时任务
- 与单服务器轻量部署匹配
- 报告查看后标记为"已查看"，不再重复弹出

### 6.2 报告内容

| 字段 | 说明 |
|------|------|
| 各精力等级任务数 | 高 X 个 / 中 Y 个 / 低 Z 个 |
| 消耗精力点合计 | daily_load 值 |
| 每日预算 | daily_budget 值 |
| 完成率 | 已完成任务数 / 总任务数 × 100% |
| 未完成原因统计 | 精力不足 X / 时间不够 Y / 被打断 Z |

## 检查清单

各角色实现涉及精力管理的功能后应自查：

- [ ] energy_level 枚举为 HIGH/MEDIUM/LOW，energy_points 正确映射（4.0/2.0/1.0）
- [ ] 边界归属正确：恰好 2h → 中，恰好 4h → 高
- [ ] daily_budget 默认 8.0，可自定义 4.0-16.0 步长 0.5
- [ ] weekly_budget 默认 56.0（8.0×7天），覆盖完整 7 天
- [ ] 周末任务与工作日同等处理，energy_points 正常计入
- [ ] daily_load = 当日全部时段任务 energy_points 之和
- [ ] weekly_load = 当周（周一至周日）所有任务 energy_points 之和
- [ ] overload_threshold = weekly_budget，≥ 阈值触发预警
- [ ] 任务层级严格 3 层，仅 level=3 可排程
- [ ] 每日精力消耗报告在当日首次打开时自动弹出（前一天未查看的报告）
