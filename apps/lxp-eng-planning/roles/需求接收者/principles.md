# 需求接收者 原则

## 设计原则

1. **七大要素完整**：需求文档必须包含目标、核心能力、角色清单、协作流程、对抗性要求、验收标准、文档层级七大要素，缺一不可
2. **验收标准可证伪**：每条验收标准必须有具体的验证方式和通过/不通过判定条件，不可使用模糊表述（如"体验良好""代码规范"）
3. **角色清单与拓扑一致**：角色清单中定义的角色必须与协作流程描述的 DAG 拓扑一致，无错位、无遗漏、无多余
4. **核心能力可编号**：每项核心能力必须有功能编号（如 F1、F2）和优先级标注（P0/P1/P2），不可使用无编号的散列描述
5. **协作流程可追溯**：协作流程必须描述主链拓扑、迭代循环规则、流程类型标注（线性/并行/迭代），下游角色可据此理解执行顺序
6. **文档层级有依赖**：文档层级要素必须列出文档类型列表、文档间依赖关系、文档层与执行层关系、每层审阅角色

## 校验清单

- [ ] 需求文档包含七大要素（目标/核心能力/角色清单/协作流程/对抗性要求/验收标准/文档层级），无遗漏
- [ ] 每条验收标准可证伪（有验证方式 + 判定标准 + 期望值）
- [ ] 核心能力每项有功能编号和优先级标注
- [ ] 角色清单 ≥ 2 个角色，且角色名、类型、职责、producer 入口标识齐全
- [ ] 角色清单与协作流程描述的 DAG 拓扑一致，无错位
- [ ] 协作流程描述了主链拓扑 + 迭代循环规则 + 流程类型标注
- [ ] 对抗性要求明确了对抗角色和对抗维度
- [ ] 文档层级列出了文档类型列表及文档间依赖关系
