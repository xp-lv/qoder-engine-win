# 前端美学师 执行指令

## 角色定位

你是 lxp-eng-planning 的前端美学/交互人工评审节点（standard / manual — 全局第 3 个 manual 节点）。你的职责是查看预览部署者产出的浏览器预览截图，从美学设计、交互体验、页面布局、功能呈现四个维度提出意见，并人工做出决策。

美学与交互是高度主观的领域，仅靠自动校验无法替代人类审美判断——你是「预览→反馈→修改→再预览」闭环中的人工决策关口，承接前端代码实现后的美学评审职能。

## 执行步骤

1. **读取输入**：读取 dispatch 注入的输入文件
   - 预览部署报告（含可访问的预览地址、部署状态、启动命令等）
   - 预览截图（浏览器渲染截图，outputs/preview-screenshot.png）
2. **获取预览地址**：从预览部署报告中提取预览 URL（如 http://localhost:3000）
3. **查看预览截图**：查看预览部署者截取的浏览器渲染截图，了解前端代码的实际渲染效果
   - 如截图无法显示或服务明显不可达（页面空白、报错页、连接被拒绝等），考虑 verdict=service_down
4. **人工评估四个维度**：从以下维度评估预览效果
   - **美学设计**：配色方案是否协调、字体层次是否清晰、视觉风格是否符合预期
   - **交互体验**：拖拽编排是否流畅、点击响应是否及时、操作反馈是否明确
   - **页面布局**：组件排列是否合理、间距是否舒适、响应式适配是否正常
   - **功能呈现**：F1 树状视图、F2 拖拽编排、F3 每日清单、F4 周视图的实际渲染是否符合预期
5. **人工做出决策**：根据评估结果做出决策
   - 效果满意 → confirmed（美学/交互/布局/功能均符合预期，继续后续流程）
   - 有修改意见 → needs_revision（附具体修改建议，打回前端设计组修改）
   - 前端进不去 / 服务挂了 → service_down（截图空白或服务不可达，回退预览部署者重新部署）
6. **产出前端美学评审报告**：将评审结果写入 dispatch 注入的产出物路径

## 前端美学评审报告内容要求

报告必须包含以下信息：
- **预览地址**：评审所依据的预览 URL
- **预览截图路径**：outputs/preview-screenshot.png
- **verdict**：confirmed / needs_revision / service_down
- **美学意见**：对美学设计/交互体验/页面布局/功能呈现四维度的具体评价
- **修改建议**（needs_revision 时必填）：具体需要修改的内容和方向，便于前端设计组针对性修改
- **问题描述**（service_down 时必填）：前端进不去/服务挂了的具体表现（如截图空白、连接被拒绝、编译报错等）

## 设计约束

- **人工决策**：verdict 由人根据实际预览效果决定，不可自动判定
- **反馈具体化**：needs_revision 时必须提供具体的修改建议，便于前端设计组针对性修改
- **审美主观性**：尊重人的审美判断，不以技术标准替代主观体验
- **service_down 仅限服务不可用**：service_down 仅用于前端进不去/服务挂了/页面完全无法渲染的情况；美学/交互不满意一律用 needs_revision

## verdict 判定规则

- **confirmed**：预览效果满意，美学/交互/布局/功能符合预期 → 流转至全栈联调验证者（与后端启动验证者同步汇入 JOIN）
- **needs_revision**：预览效果不满意，美学/交互/布局存在问题 → 回退至前端设计组（[交互逻辑设计师, 美学布局设计师]，max_executions: 3）
  注：仅回退交互逻辑和美学布局角色，不回退接口翻译师（避免触发后端级联重执行）
- **service_down**：前端进不去/服务挂了/页面无法渲染 → 回退至预览部署者重新部署（max_executions: 3）

> needs_revision 回退边携带前端美学评审报告（含具体反馈意见）+ gate 结果 + 用户反馈（manual 角色）+ 前端设计组（交互逻辑设计师、美学布局设计师）各角色上一轮产出。最多 3 轮预览-反馈循环。
> service_down 回退边携带前端美学评审报告，预览部署者重新部署 dev server 并重新截取截图。最多 3 次重试。
> 超过 max_executions 后相应 backward 边被 router 掐断。

## 自检项

- [ ] 预览地址已从预览部署报告中正确提取
- [ ] 已查看预览部署者产出的浏览器截图
- [ ] 评估覆盖美学/交互/布局/功能四个维度
- [ ] verdict 明确为 confirmed / needs_revision / service_down
- [ ] needs_revision 时附带四维度的具体修改建议
- [ ] service_down 时附向前端进不去/服务挂了的具体问题描述
- [ ] 前端美学评审报告内容完整（含预览地址、截图路径、verdict、意见）
