# 回归验证标准手册 — 变更后引擎正确性验证

> **本文档用途**：支撑回归验证者产出验证报告（AC-9/AC-10/AC-11）和架构红队 6 维对抗（AC-13）。定义验证变更后引擎正确性的标准检查项和检查方法。

---

## 1. 目的

本文档定义代码变更后引擎正确性验证的标准检查项、检查方法和通过/失败判据。回归验证者和架构红队按本手册的检查项逐项验证，确保变更不破坏引擎现有功能。

## 2. 适用角色

- **回归验证者**：按本手册执行编译正确性、DAG 可达性、行为兼容性、Gate 校验兼容性验证
- **架构红队**：按本手册的 6 维对抗检查清单执行极限测试

## 3. 编译正确性验证（AC-9）

### 3.1 验证方法

执行 compiler.py 编译链路检查：
```bash
python engine/scripts/compiler.py --check <app.yaml路径>
```

### 3.2 检查项

| 检查项 | 通过条件 | 失败条件 |
|---|---|---|
| 语法正确性 | app.yaml 语法合法 | YAML 语法错误 |
| 角色定义完整性 | 所有角色有 outputs | 有角色缺少 outputs |
| edges 格式 | 所有边格式合法 | 边格式错误 |
| verdict 提取 | when: 表达式正确 | when: 表达式语法错误 |
| producer 展开 | 校验角色正确生成 | 校验角色生成失败 |
| knowledge 注入 | inject_to 引用的角色存在 | inject_to 引用不存在的角色 |

### 3.3 错误分类

| 错误类型 | 严重级别 | 示例 |
|---|---|---|
| 语法错误 | critical | YAML 格式错误 |
| 类型错误 | critical | 必填字段缺失 |
| 依赖缺失 | major | inject_to 引用的角色不存在 |
| 警告 | minor | 冗余字段 |

### 3.4 通过判据

- 编译无 critical 和 major 错误 → PASS
- 编译有 critical 或 major 错误 → FAIL

## 4. DAG 可达性验证（AC-10）

### 4.1 正向遍历（从入口可达）

从 producer 入口出发，沿 edges 遍历所有角色：
```
入口 → confirmed → 角色 A → confirmed → 角色 B → ... → 完成
```

**通过条件**：所有角色从入口可达
**失败条件**：存在不可达角色（孤立角色）

### 4.2 反向遍历（可达终态）

从每个角色出发，沿 edges 反向遍历，检查是否可达完成终态：
```
角色 X → ... → 完成
```

**通过条件**：所有角色可达完成终态
**失败条件**：存在无法到达终态的角色（死胡同）

### 4.3 环检测

检查 DAG 中是否存在循环：
```
A → B → C → A  ← 循环！
```

**通过条件**：DAG 无环
**失败条件**：存在循环依赖

### 4.4 孤立角色检测

检查是否有角色无入边或无出边（非终态角色）：
```
角色 X（无入边）← 孤立！
角色 Y（无出边，非终态）← 死胡同！
```

### 4.5 通过判据

- 正向遍历 + 反向遍历 + 环检测 + 孤立检测全部通过 → PASS
- 任一检测失败 → FAIL

## 5. verdict 出边完备性验证

### 5.1 检查方法

对每个角色：
1. 读取 schema.json 的 result.verdict.enum（该角色所有可能的 verdict 值）
2. 读取 ROUTER.json 中该角色的 transitions（所有出边）
3. 检查 enum 中的每个 verdict 是否有对应出边

### 5.2 检查项

| 检查项 | 通过条件 | 失败条件 |
|---|---|---|
| verdict 出边覆盖 | enum 中每个 verdict 有对应 transition | 有 verdict 无出边（悬空） |
| JOIN 角色特殊处理 | JOIN 边不提取 verdict，但保留 enum | JOIN 边错误提取 verdict |
| fail 排除 | schema enum 中无 fail | schema enum 中有 fail |

### 5.3 通过判据

- 所有角色的 verdict 出边完备，无悬空 verdict → PASS

## 6. 行为兼容性验证（AC-11）

### 6.1 快照对比方法

变更前后对同一 app.yaml 执行编译，对比编译产物：
```
编译前快照 → ROUTER_before.json + registry_before.json
编译后快照 → ROUTER_after.json + registry_after.json
对比差异 → 预期差异 vs 意外差异
```

### 6.2 输入输出对比方法

对关键函数，变更前后用相同输入执行，对比输出：
```
输入 X → 变更前函数 → 输出 Y_before
输入 X → 变更后函数 → 输出 Y_after
Y_before == Y_after? → 预期一致 or 意外差异
```

### 6.3 状态迁移对比方法

变更前后用相同操作序列，对比 STATE.json 变化：
```
操作序列 → 变更前 STATE_before.json
操作序列 → 变更后 STATE_after.json
对比 → 预期一致 or 意外差异
```

### 6.4 向后兼容性分析框架

| 兼容性维度 | 检查方法 | 通过条件 |
|---|---|---|
| API 签名兼容 | 对比变更前后的函数签名 | 签名不变或有默认参数新增 |
| 数据格式兼容 | 对比变更前后的数据结构 | 旧数据可被新代码正确解析 |
| 配置兼容 | 对比变更前后的配置格式 | 旧配置文件可被新代码读取 |
| 编译产物兼容 | 对比 ROUTER.json/registry.json 格式 | 格式版本兼容 |

### 6.5 通过判据

- 所有行为对比无意外差异，兼容性分析完整 → PASS
- 存在未标注的不兼容行为 → FAIL

## 7. Gate 校验兼容性验证

### 7.1 检查方法

| 检查项 | 方法 | 通过条件 |
|---|---|---|
| 产出物格式 | 变更后各角色产出物通过 schema.json 校验 | 所有角色 PASS |
| schema 一致性 | schema.json 与角色实际输出一致 | 字段/类型/enum 一致 |
| return-package 格式 | return-package.schema.json 校验通过 | 格式合规 |

## 8. 对抗性测试 6 维标准检查清单（AC-13）

### 8.1 边界条件

| 测试用例 | 测试方法 | 通过条件 |
|---|---|---|
| 空输入 | 传入空字符串/空列表/None | 不崩溃，有合理处理 |
| 超长输入 | 传入极大字符串/列表 | 不崩溃，不超时 |
| 极端值 | 传入负数/零/最大值 | 行为正确 |
| 特殊字符 | 传入 Unicode/控制字符 | 正确处理 |
| 越界访问 | 传入超出范围的索引 | 有异常处理 |

### 8.2 向后兼容性

| 测试维度 | 测试方法 | 通过条件 |
|---|---|---|
| 现有 APP 运行 | 变更后编译运行现有 APP | 正常运行 |
| API 调用方 | 搜索所有调用受影响 API 的代码 | 兼容性分析完整 |
| 数据格式 | 旧格式数据 + 新代码 | 正确解析 |

### 8.3 新缺陷引入

| 检查项 | 检查方法 | 通过条件 |
|---|---|---|
| 逻辑错误 | diff 审查变更代码 | 无逻辑错误 |
| 资源泄漏 | 检查文件句柄/内存管理 | 无泄漏 |
| 竞态条件 | 检查并发访问路径 | 无竞态 |
| 异常处理 | 检查 try/except 覆盖 | 无遗漏 |

### 8.4 DAG 完整性

| 检查项 | 检查方法 | 通过条件 |
|---|---|---|
| 可达性 | 正向+反向遍历 | 全部可达 |
| 无环 | 拓扑排序 | 无循环 |
| 无孤立 | 入边/出边检查 | 无孤立节点 |

### 8.5 状态机一致性

| 检查项 | 检查方法 | 通过条件 |
|---|---|---|
| step_status 语义 | 与 SDK_SPEC §5 对齐 | 一致 |
| finished 消除 | dispatch 后清除 | 正确清除 |
| edge_counts 递增 | 每次执行递增 | 正确递增 |
| verdict 驱动 | verdict→状态变更映射 | 正确映射 |

### 8.6 回滚安全性

| 检查项 | 检查方法 | 通过条件 |
|---|---|---|
| 回滚步骤完整 | 每个变更项有回滚步骤 | 全部覆盖 |
| 回滚顺序正确 | 先回滚依赖项 | 顺序正确 |
| 回滚后编译 | 回滚后执行编译 | 编译通过 |
| 回滚后运行 | 回滚后执行工作流 | 正常运行 |

## 9. 检查清单

回归验证者和架构红队使用以下总检查清单：

- [ ] 编译正确性已验证（AC-9）
- [ ] DAG 可达性已验证（正向+反向+环+孤立）（AC-10）
- [ ] verdict 出边完备性已验证
- [ ] 行为兼容性已验证（快照/IO/状态对比）（AC-11）
- [ ] Gate 校验兼容性已验证
- [ ] 边界条件 6 类测试已执行
- [ ] 向后兼容性 3 维测试已执行
- [ ] 新缺陷引入 4 类检查已执行
- [ ] DAG 完整性 3 项检查已执行
- [ ] 状态机一致性 4 项检查已执行
- [ ] 回滚安全性 4 项检查已执行
- [ ] 所有 findings 标注严重级别
