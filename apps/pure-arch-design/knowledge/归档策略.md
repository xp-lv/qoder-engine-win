# 归档策略

> 跨层重跑时保证 AC-4（severity 变化链可追溯）的归档机制。
> 架构师和终审共享遵守。

## 为什么需要归档

跨层重跑（consistency_defect 或 reqid_approved 后重跑全部 3 轮）时，新一轮红队和架构师会**覆盖性写入固定路径**（outputs/架构蓝图.md、outputs/R{1,2,3}-问题清单.json、outputs/R{1,2,3}-响应记录.json）。

如果不归档，前次对抗记录丢失，违反 AC-4"severity 变化链可追溯"。

## 归档目录结构

```
outputs/
├── archive/
│   ├── run-1/                         ← 第 1 次跨层重跑归档
│   │   ├── 架构蓝图.md
│   │   ├── R1-问题清单.json
│   │   ├── R1-响应记录.json
│   │   ├── R2-问题清单.json
│   │   ├── R2-响应记录.json
│   │   ├── R3-问题清单.json
│   │   ├── R3-响应记录.json
│   │   └── 终审裁决书.json
│   ├── run-2/                         ← 第 2 次归档（如发生）
│   │   └── ...
│   └── ...
├── 架构蓝图.md                         ← 当前轮（最新状态）
├── R1-问题清单.json
├── R1-响应记录.json
└── ...
```

## 归档时机

**触发条件**：以下两种场景之一发生时，架构师**必须**在覆盖固定路径前先归档：

1. 收到 `consistency_defect` 回退（终审发现全局一致性缺陷，重跑全部 3 轮）
2. 收到 `reqid_approved` 后的 L2 更新链路（REQ-ID 增补后重跑）

## 归档执行者

**架构设计师主动执行**（skill 指引强制）。

终审**不主动归档**，但可读取 `outputs/archive/` 做跨轮一致性对比。

## run-{N} 计数规则

- N 为**全局跨层重跑轮次计数**，从 1 开始
- 第 1 次跨层重跑归档到 `run-1`
- 第 2 次归档到 `run-2`
- 依此类推
- N 不随轮内迭代递增（轮内迭代不跨层）

## SDK 约束（为什么不用 {iter} 模板）

SDK carries 不支持模板替换（如 `outputs/R{n}-问题清单.json`）——SDK §4.1 carries 是字面量路径。

所以：
- **固定路径**（`outputs/R1-问题清单.json`）作为当前轮产出 + Gate 物理存在性校验基础
- **归档目录**（`outputs/archive/run-{N}/`）由架构师主动 cp 到该路径

## AC-4 可追溯性的保证方式

```
当前轮固定路径 = 最新状态（红队/架构师读写这里）
outputs/archive/run-{N}/ = 历史轮次（终审做跨轮对比时读这里）

两者互补 → AC-4 severity 变化链可追溯
```

## 归档操作步骤（架构师执行）

```
# 检测到跨层重跑触发（收到 consistency_defect 或 carries 含 REQ-ID增补审批.json）
# → 在覆盖固定路径前：

mkdir -p outputs/archive/run-{当前 N}
cp outputs/架构蓝图.md outputs/archive/run-{N}/
cp outputs/R1-问题清单.json outputs/archive/run-{N}/
cp outputs/R1-响应记录.json outputs/archive/run-{N}/
cp outputs/R2-问题清单.json outputs/archive/run-{N}/
cp outputs/R2-响应记录.json outputs/archive/run-{N}/
cp outputs/R3-问题清单.json outputs/archive/run-{N}/
cp outputs/R3-响应记录.json outputs/archive/run-{N}/
cp outputs/终审裁决书.json outputs/archive/run-{N}/  # 如存在

# 归档完成后，再覆盖固定路径产出新一轮
```

## 自检清单（架构师归档时）

- [ ] 是否检测到跨层重跑触发条件？
- [ ] N 计数是否正确（递增不跳号）？
- [ ] 是否归档了全部 8 类产出（蓝图 + 3 问题清单 + 3 响应记录 + 终审裁决书）？
- [ ] 是否在覆盖固定路径前完成归档？

## 自检清单（终审跨轮对比时）

- [ ] 是否读取 `outputs/archive/run-{N}/` 做跨轮对比（如归档目录存在）？
- [ ] severity 变化链是否可追溯（当前轮固定路径 + archive 历史）？
