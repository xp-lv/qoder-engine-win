# 需求描述

## 目标
设计一个多角色 app，用于自动生成技术文档族。

## 角色需求
- **资料收集者**：读取用户提供的源材料（PDF/URL/Markdown），提取关键信息
- **大纲设计者**：根据收集的资料设计文档结构大纲
- **内容撰写者**：按大纲逐章节撰写内容
- **质量审阅者**：审阅文档完整性和准确性

## 流程要求
- 资料收集 → 大纲设计 → 内容撰写 → 质量审阅
- 审阅者如果不满意，可以退回内容撰写者重写（最多3次）
- 大纲设计者需要人工确认后才推进
- 资料收集者产出物为文档，其他为 JSON 报告

## 其他约束
- 大纲设计者使用 manual confirm
- 其他角色使用 auto confirm
- 内容撰写者为 producer 类型（自动展开校验）
